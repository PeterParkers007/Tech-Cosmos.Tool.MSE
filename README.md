## 📖 这是什么？

在 Unity 开发中，我们经常会写出这样的代码：

```csharp
// ❌ 魔法字符串 — 拼写错误要到运行时才发现
gameObject.tag = "Player";
animator.SetTrigger("Jump");
SceneManager.LoadScene("MainMenu");
```

`"Player"`、`"Jump"`、`"MainMenu"` —— 这些就是所谓的**魔法字符串**（Magic Strings）。它们散落在代码各处，难以统一管理，极易拼写错误，而且 IDE 无法提供智能提示，往往要到运行时才能发现问题。

**Magic String Eliminator** 就是来消灭它们的！

它提供了一个可视化的 Editor 窗口，你可以像填表一样定义字符串常量，然后一键生成**强类型的枚举代码**。从此：

```csharp
// ✅ 编译时安全 — 拼写错误当场报错，IDE 自动补全
gameObject.tag = GameTags.Player.GetString();
animator.SetTrigger(AnimationParams.Jump.GetString());
SceneManager.LoadScene(SceneNames.MainMenu.GetString());
```

## ✨ 核心功能

| 功能 | 说明 |
|------|------|
| 📝 **可视化编辑** | 在 Editor 窗口中直接添加/编辑键值对，所见即所得 |
| 📊 **批量表格模式** | 像 Excel 一样批量编辑大量条目，支持一键添加多行 |
| 🔗 **键值同步** | 一键开启 Key = Value 模式，适用于 Tags、Scenes、Layers 等场景 |
| 📁 **命名空间分组** | 右侧面板按命名空间自动分类展示已生成的所有枚举 |
| ✏️ **编辑已生成文件** | 直接编辑已有的枚举文件，覆盖保存，无需手动修改代码 |
| 📥📤 **CSV 导入导出** | 支持从 CSV 文件批量导入条目，也可导出为 CSV 备份 |
| 💾 **偏好记忆** | 关闭窗口后保留编辑模式、命名空间、文件名等所有设置 |

## 🚀 快速开始

### 环境要求

- Unity 2019.4 或更高版本
- 无需额外依赖

### 安装

<details>
<summary><b>方式一：通过 Package Manager（推荐）</b></summary>

1. 在 Unity 中打开 `Window > Package Manager`
2. 点击左上角 `+` → `Add package from git URL...`
3. 输入仓库地址：
   ```
   https://github.com/PeterParkers007/Tech-Cosmos.Tool.MSE.git
   ```
4. 点击 `Add` 等待导入完成

</details>

<details>
<summary><b>方式二：手动安装</b></summary>

1. 克隆或下载本仓库
2. 将整个文件夹放入项目的 `Packages/` 目录下
3. 或者放入 `Assets/` 目录下（确保在 `Editor` 文件夹中）

</details>

### 基本使用

1. 打开工具窗口：`Tools > Magic String Eliminator`
2. 设置 **枚举名称**（Enum Name）和 **命名空间**（Namespace）
3. 选择编辑模式：
   - **Single**：逐个添加键值对
   - **Batch Table**：批量表格编辑
4. 添加你的字符串条目
5. （可选）开启 🔗 **Key=Value** 模式，让值自动跟随键名
6. 点击 **Generate Enum** 生成代码
7. 在代码中通过 `using YourNamespace;` 引入并使用生成的枚举

## 📖 生成的代码示例

假设你定义了以下条目：

| Key | Value |
|-----|-------|
| Player | Player |
| Enemy | Enemy |
| Coin | Collectible |

工具会在 `Assets/Generated/Enums/` 目录下自动生成代码：

````csharp
// This file is auto-generated by Magic String Eliminator. Do not modify manually.
// Generated at: 2026-04-29 18:15:10

using System.Collections.Generic;

namespace GameTags
{
    public enum Tags
    {
        Player,
        Enemy,
        Coin
    }

    public static class TagsExtensions
    {
        private static readonly Dictionary<int, string> _stringMap =
            new Dictionary<int, string>
            {
                { (int)Tags.Player, "Player" },
                { (int)Tags.Enemy, "Enemy" },
                { (int)Tags.Coin, "Collectible" }
            };

        public static string GetString(this Tags value)
        {
            return _stringMap.TryGetValue((int)value, out var result) ? result : value.ToString();
        }

        public static Tags FromString(string value)
        {
            foreach (var pair in _stringMap)
            {
                if (pair.Value == value)
                    return (Tags)pair.Key;
            }
            throw new System.ArgumentException($"No Tags with string value '{value}'");
        }
    }
}
````

### 在代码中使用

```csharp
using GameTags;

public class Example : MonoBehaviour
{
    void Start()
    {
        // 获取字符串值
        string playerTag = Tags.Player.GetString();
        gameObject.tag = playerTag;
        
        // 从字符串转换回枚举
        Tags tag = Tags.FromString("Enemy");
        
        // 直接比较（编译时类型安全）
        if (tag == Tags.Coin)
        {
            Debug.Log("Found a coin!");
        }
    }
}
```

## ❓ 常见问题

<details>
<summary><b>生成的文件可以修改吗？</b></summary>

不建议直接手动修改生成的文件。如需修改，请使用工具右侧面板的 **Edit** 按钮进入编辑模式，修改后点击 **Save Changes** 覆盖保存。

</details>

<details>
<summary><b>生成的文件在构建中会包含吗？</b></summary>

会包含。生成的是普通的 C# 枚举代码，属于项目的一部分。工具本身位于 `Editor` 目录，不会包含在最终构建中。

</details>

<details>
<summary><b>如何修改生成文件的保存路径？</b></summary>

目前生成路径固定为 `Assets/Generated/Enums/`。如需修改，可在 `GenerateEnumFile()` 方法中调整 `directory` 变量。

</details>

<details>
<summary><b>支持哪些 Unity 版本？</b></summary>

最低支持 Unity 2019.4。理论上也兼容更早版本，但未经过测试。

</details>
